# Paper Summary & Review Skill

## 功能描述
对学术论文进行深度总结和医学专用多维度评估，生成结构化的 summary.md 和 scores.md。

## 论文去重检查流程

### 步骤0: 检查论文是否已评估

在开始评估前，**必须**先检查论文是否已被评估：

```bash
# 检查已评估论文列表
cat workspace/papers/evaluated_papers.json
```

**去重策略**：
1. **arXiv ID 去重**：唯一标识，如果 arXiv ID 已存在，跳过评估
2. **标题完全匹配去重**：同一论文的不同版本
3. **标准化标题匹配**：保留版本标识符（++、-2、-3等），避免系列论文误判

**重要说明**：
- Transolver、Transolver++、Transolver-3 是三篇不同的论文
- 版本标识符（++、-数字）不会被移除，避免误判

如果论文已存在：
- 输出提示：`论文 [标题] (arXiv: [ID]) 已于 [日期] 评估，跳过重复评估`
- 不执行后续步骤

如果论文未评估：
- 继续执行数据获取流程
- 评估完成后，将论文信息添加到 `evaluated_papers.json`

## 数据获取流程

### 核心原则
> Semantic Scholar 数据只允许调用一次。
> 后续所有评分计算必须基于第一次获取的数据。
> 禁止在影响力计算阶段再次调用 API。

### Semantic Scholar 数据集成

在进行论文评估前，**必须**先通过 Semantic Scholar API 获取完整metadata，包含准确的引用数据和作者信息：

#### 步骤1: 获取论文数据

```bash
# 通过 arXiv ID 获取（推荐，最准确）
python skills/semantic-scholar/semantic_scholar_api.py paper-by-arxiv "arxiv-id" --format json > metadata.json

# 通过标题搜索
python skills/semantic-scholar/semantic_scholar_api.py paper-by-title "paper-title" --format json > metadata.json
```

### 输出要求

必须保存为：`metadata.json`

该文件将作为后续所有评分计算的唯一数据来源

#### 步骤2: 从 metadata.json 提取字段

必须提取以下字段：

| 字段 | 用途 | 说明 |
|------|------|------|
| `citationCount` | 影响力评估 | 论文被引用次数 |
| `publicationDate` | Date-Citation计算 | 论文发表日期 |
| `year` | 论文年龄计算 | 发表年份 |
| `authors` | 作者影响力 | 作者列表和ID |
| `venue` / `journal` | 发表场所评估 | 会议/期刊名称 |
| `openAccessPdf.url` | PDF下载 | 开放获取链接 |

#### 步骤3: 写入 summary.md 基本信息

将获取的数据记录到 `summary.md` 的基本信息部分：

```markdown
## 基本信息
- **标题**: [论文标题]
- **作者**: [作者列表]
- **发表时间**: ...
- **arXiv ID**: ...
- **Semantic Scholar ID**: ...
- **引用数量**: number (截至 当前日期)
- **发表场所**: ...
- **PDF路径**: [本地PDF路径]
- **检索日期**: [当前日期]
```

## 医学专用评分模板（新增）

> 不再只使用通用四维。医学场景默认在以下两套模板中二选一。

### 模板 A：临床医学论文（clinical_medicine_v1）

维度（每项 1-10 分）：
1. 临床问题重要性
2. 研究设计强度
3. 样本量与代表性
4. 偏倚风险
5. 统计分析合理性
6. 临床可推广性
7. 指南/实践价值

### 模板 B：生物信息学论文（bioinformatics_v1）

维度（每项 1-10 分）：
1. 数据来源可靠性
2. 队列规模与外部验证
3. 生物学解释力度
4. 方法新颖性
5. 可复现性（代码/流程）
6. 过拟合风险
7. 转化潜力

### 医学扣分项（按需触发）

- 只有单数据库、没外部验证 → 扣分
- ROC/AUC 好看但机制弱 → 扣分
- 只做分子对接/MD，没有细胞或组织验证 → 扣分
- 生存分析显著，但混杂校正不足 → 扣分
- 泛癌分析很多，但主线不聚焦 → 扣分

具体规则见：`skills/paper-review/templates.medical_scoring.json`

### 评分计算（医学版）

```text
base_score = mean(all_dimension_scores)
penalty_score = sum(triggered_penalties)
adjusted_base = max(1.0, base_score - penalty_score)
final_score = adjusted_base * 0.9 + impact_score * 0.1
```

## 评估完成后的操作

### 更新已评估论文列表

评估完成后，**必须**使用以下命令安全地更新 `evaluated_papers.json`（已处理并发锁和去重检查）：

```bash
python skills/paper-review/scripts/update_registry.py \
  --id "[arXiv ID]" \
  --title "[完整论文标题]" \
  --short_title "[简短标题]" \
  --score "[最终评分]"
```

**示例**：
```bash
python skills/paper-review/scripts/update_registry.py \
  --id "2401.12345" \
  --title "Geometry-Aware Neural Operator for 3D Mesh" \
  --short_title "GANO-3D" \
  --score "8.25"
```

**注意事项**：
- 脚本会自动检查论文是否已存在（基于 arXiv ID 和标题去重）
- 使用文件锁防止并发写入冲突
- 如果论文已存在，会跳过并提示

## 评分标准详细说明

### 临床医学模板评分标准（7维）

1. **临床问题重要性**：疾病负担、未满足需求、结局改善潜力
2. **研究设计强度**：RCT > 前瞻队列 > 回顾队列 > 病例对照 > 横断面
3. **样本量与代表性**：样本量、中心数、人群覆盖、纳排标准合理性
4. **偏倚风险**：选择偏倚、信息偏倚、混杂偏倚控制情况
5. **统计分析合理性**：模型设定、假设检验、多重校正、敏感性分析
6. **临床可推广性**：外部可迁移性、可实施性、成本与依从性
7. **指南/实践价值**：是否可能影响临床路径、指南或共识

### 生物信息学模板评分标准（7维）

1. **数据来源可靠性**：数据库/样本来源是否可信、批次效应处理是否合理
2. **队列规模与外部验证**：训练验证划分是否规范、是否有独立外部队列
3. **生物学解释力度**：机制链条是否成立，是否有功能验证支撑
4. **方法新颖性**：相较主流 pipeline 的实质创新程度
5. **可复现性**：代码、参数、流程图、随机种子、版本信息是否完整
6. **过拟合风险**：模型复杂度、特征泄漏、校准曲线与稳定性
7. **转化潜力**：能否落地至诊断/预后/治疗决策

### 医学专属扣分规则（建议）

- 单数据库且无外部验证：-0.5 ~ -0.7
- ROC/AUC 高但机制解释弱：-0.3 ~ -0.5
- 仅 docking/MD 无细胞或组织验证：-0.3 ~ -0.5
- 生存分析显著但混杂校正不足：-0.3 ~ -0.5
- 泛癌分析堆砌但主线不聚焦：-0.2 ~ -0.4

### 影响力评分标准与 Date-Citation 权衡机制

#### 基础影响力评分标准

| 分数 | 描述 |
|------|------|
| 9-10 | 解决重大问题，具有广泛科研和应用价值，显著超越业界前沿 |
| 7-8 | 解决重要问题，有明确应用价值，达到或略超业界前沿 |
| 5-6 | 解决一般问题，有一定应用价值，接近业界水平 |
| 3-4 | 问题价值有限，应用前景不明确 |
| 1-2 | 问题价值很小，缺乏应用前景 |

#### Date-Citation 权衡机制

**设计目标**：公平对比不同发表时间论文的影响力

**调整规则**：

| 论文年龄 | 引用情况 | 调整因子 |
|---------|---------|---------|
| ≤ 3个月 | - | +0.2 |
| 3-24个月 | 引用数 ≥ 50 | +0.5 |
| 3-24个月 | 引用数 20-49 | +0.3 |
| 3-24个月 | 引用数 10-19 | +0.2 |
| 3-24个月 | 引用数 < 10 | +0.1 |
| > 24个月 | 引用数 ≥ 200 | +0.5 |
| > 24个月 | 引用数 100-199 | +0.4 |
| > 24个月 | 引用数 50-99 | +0.3 |
| > 24个月 | 引用数 20-49 | +0.2 |
| > 24个月 | 引用数 < 20 | +0.0 |

**引用密度奖励**（适用于所有年龄段）：
- 引用密度 ≥ 10次/月：额外 +0.2
- 引用密度 5-10次/月：额外 +0.1

**调整因子上限**：+1.0（最终影响力评分不超过10分）
